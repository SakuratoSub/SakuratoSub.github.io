命名规范

发表于 2021-04-28 | 分类于 Fansub

Sakurato作品命名格式标准

标签

格式化命名通常会采用以下几种标签组成

Tag
组名或合作组名
例如[Sakurato]、[XXX&Sakurato]

CHS_Title
番剧简体中文译名,以萌番组Tag为主

CHT_Title
番剧繁体中文译名,以萌番组Tag为主

Main_Title
番剧罗马音，以AniDB中番剧页面的Main Title为主

Episode
当前发布集数，以两位罗马数字组成，不足部分前面补0
对于单集视频重置版，在集数后面标上版本，以小写v+一位罗马数字组成，最初版省略
例如[01]、[12]、[03v2]

Release_Version
当前发布版本
合集版本使用Fin
重置版使用Rev

Video_Format
视频流格式，以视频流编码-位深组成，
例如[AVC-8bit]、[HEVC-10bit]

Screen
视频流分辨率和刷新率，分辨率由分辨率的数字加小写字母p组成，非24000/1001帧率视频流可以省略刷新率参数
例如[720p]、[1080p]、[1080p@60FPS]、[1440p]、[2160p]

Audio_Format
音频流格式,以FLAC、AAC两种为主
例如[FLAC]、[AAC]

Language_en
字幕语言格式，主要有简体中文：CHS、繁体中文：CHT、日文：JPN三种，多语言版本之间用&连接
例如[CHS]、[CHS&CHT]、[CHS&JPN]

Vide_Version
字幕语言格式，主要有简体中文：CHS、繁体中文：CHT、日文：JPN三种，多语言版本之间用&连接
例如[CHS]、[CHS&CHT]、[CHS&JPN]

Release_Time
视频发布时间，以AniDB中番剧页面的Year为主，格式为年（4位数字） 月（2位数字） 日（2位数字）
例如[20200101]代表2020年01月01日

Composition
项目内容组成结构，主要有TVRip、BDRip、OVA、OAD、SPs、CDs，多种组合之间用+连接。
注意：当存在SPs时若同时包含OVA或OAD，OVA或OAD可省略
例如[TVRip+OVA+OAD]、[BDRip+SPs+CDs]

Content
其他文件打包标识符，主要有Subtitle、Fonts、Chapter等

组成

单视频文件名构成

[Tag] Main_Title [Episode][Video_Format Screen Audio_Format][Language].mp4/mkv
注意：

  • Main_Title和Screen的前后各有一个空格分隔
  • Video_Format Screen Audio_Format三个Tag用一个[]包起来

案例

合集文件夹名构成

[Tag][Release_Time] Main_Title [Episode Release_Version][Video_Format Screen Audio_Format][Language]
注意：

  • Main_Title和Screen的前后各有一个空格分隔
  • Video_Format Screen Audio_Format三者使用一个[]包起来
  • Episode和Release_Version之间有一个空格，两者使用一个[]包起来

案例

其他文件命名构成

字幕

[Tag] Main_Title [Episode].Language.ass
案例

其他文件打包构成

[Tag] Main_Title [Content].7z
案例

发布页

[Tag] CHS_Title/Main_Title [Episode][Screen][]
